Core Cloud Concepts

Simple explanations for beginners

Tenancy

A tenancy is the top-level container in cloud computing. It represents the cloud account that owns and manages all resources.

Region

A region is a geographical area where cloud data centers are located. Choosing a nearby region helps reduce latency.

Availability Domains

Availability Domains are isolated data centers within a region. They help ensure high availability and fault tolerance.

Compartments

Compartments are used to organize and isolate cloud resources. They help manage access control and resource organization.

Tags

Tags are labels applied to cloud resources. They help in identifying, tracking, and managing resources.

Quotas & Limits

Quotas and limits define the maximum number of cloud resources that can be used within a tenancy.
